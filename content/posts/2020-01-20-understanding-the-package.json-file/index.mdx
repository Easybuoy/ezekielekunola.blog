### **What is a `package.json` file**

A package.json file is a file that exists in the root of a project and it holds metadata relevant to the project and is mostly used for managing dependencies of the project, as well as scripts.

### **Properties of a package.json file**

#### name

The name property is a required field in the `package.json` file, it represents the name of the project.

```json
{
  "name": "Storemanager"
}
```

#### version

The version is also a required field in the `package.json` file. The property denotes the current version of the module for the project.

```json
{
  "version": "1.0"
}
```

#### description

The description property is used in describing the project.

```json
{
  "description": "Storemanager is a project that..."
}
```

### **Creating a `package.json` file**

There are two ways of creating a `package.json` file.

- npm or yarn
  For `npm`, run `npm init` on your terminal

  > Note: To use this method, you need to have [Node]() installed on your system.

- Manually
  To create manually, create a `package.json` file manually in the root of the project, and add an empty object.
